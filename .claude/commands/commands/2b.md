## RAG & Embeddings

### Quando usar RAG

| Caso de uso | Recomendado |
|-------------|-------------|
| Busca semântica em documentos do projeto | ✅ RAG com pgvector |
| Contexto que muda frequentemente (contratos, processos) | ✅ RAG |
| Conhecimento estático pequeno (< 20k tokens) | ✅ Injetar no contexto direto |
| Perguntas sobre dados estruturados (banco de dados) | ✅ Text-to-SQL em vez de RAG |

### Setup pgvector (PostgreSQL)

```sql
-- Habilitar extensão (Migration Prisma)
CREATE EXTENSION IF NOT EXISTS vector;

-- Tabela de embeddings
CREATE TABLE document_embeddings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  doc_id UUID REFERENCES documents(id) ON DELETE CASCADE,
  chunk_index INT NOT NULL,
  content TEXT NOT NULL,
  embedding vector(1536),  -- 1536 para text-embedding-3-small
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Índice para busca por similaridade
CREATE INDEX ON document_embeddings
  USING hnsw (embedding vector_cosine_ops);
```

### Estratégia de Chunking

```typescript
// Regra geral: chunks de 512 tokens com 50 tokens de overlap
function chunkText(text: string, chunkSize = 512, overlap = 50): string[] {
  const words = text.split(' ');
  const chunks: string[] = [];
  for (let i = 0; i < words.length; i += chunkSize - overlap) {
    chunks.push(words.slice(i, i + chunkSize).join(' '));
  }
  return chunks;
}
```

### Pipeline RAG completo

```typescript
// 1. Indexar documento
async function indexDocument(docId: string, content: string) {
  const chunks = chunkText(removePII(content));
  for (const [i, chunk] of chunks.entries()) {
    const embedding = await openai.embeddings.create({
      model: 'text-embedding-3-small',
      input: chunk,
    });
    await db.documentEmbedding.create({
      data: { docId, chunkIndex: i, content: chunk, embedding: embedding.data[0].embedding },
    });
  }
}

// 2. Buscar chunks relevantes
async function searchSimilar(query: string, topK = 5) {
  const queryEmbedding = await openai.embeddings.create({
    model: 'text-embedding-3-small',
    input: query,
  });
  // Cosine similarity via pgvector
  return db.$queryRaw`
    SELECT content, 1 - (embedding <=> ${queryEmbedding.data[0].embedding}::vector) AS similarity
    FROM document_embeddings
    ORDER BY embedding <=> ${queryEmbedding.data[0].embedding}::vector
    LIMIT ${topK}
  `;
}

// 3. Gerar resposta com contexto
async function askWithRag(question: string): Promise<string> {
  const relevantChunks = await searchSimilar(question);

  const prompt = `Contexto:\n${context}\n\nPergunta: ${question}`;

}
```

## LLM Evaluation Framework (Evals)

Antes de fazer deploy de qualquer feature de IA em produção, execute um conjunto de evals:

### Dimensões de Qualidade

| Dimensão | Como medir | Threshold mínimo |
|----------|-----------|-----------------|
| **Acurácia** | Golden dataset (input → expected output) | ≥ 85% de match exato ou semântico |
| **Hallucination rate** | Checar se resposta contém fatos não presentes no contexto | < 5% |
| **Relevância** | LLM-as-judge: "A resposta responde a pergunta?" | ≥ 90% aprovação |
| **Latência** | p95 do tempo de resposta end-to-end | < 3s para análises síncronas |
| **Custo** | Custo médio por chamada × volume projetado | Dentro do budget projetado |

### Golden Dataset

```typescript
// evals/credit-scoring.eval.ts
interface EvalCase {
  input: string;
  expectedRisk: 'low' | 'medium' | 'high';
  expectedScoreRange: [number, number];
}

const goldDataset: EvalCase[] = [
  {
    input: "cliente quitou antecipado, sem atrasos",
    expectedRisk: 'low',
    expectedScoreRange: [85, 100],
  },
  {
    input: "3 atrasos > 60 dias nos últimos 12 meses",
    expectedRisk: 'high',
    expectedScoreRange: [0, 40],
  },
];

async function runEvals() {
  let passed = 0;
  for (const testCase of goldDataset) {
    const result = await creditScoringUseCase.execute(testCase.input);
    const riskOk = result.risk === testCase.expectedRisk;
    const scoreOk = result.score >= testCase.expectedScoreRange[0] &&
                    result.score <= testCase.expectedScoreRange[1];
    if (riskOk && scoreOk) passed++;
    else console.error(`FAIL: ${testCase.input}`, result);
  }
  console.log(`Evals: ${passed}/${goldDataset.length} passed`);
  if (passed / goldDataset.length < 0.85) throw new Error('Eval threshold not met — block deploy');
}
```

## LLM Observability

Instrumentar toda chamada a LLM para depuração e custo tracking:

### Langfuse (self-hosted ou cloud)

```typescript
// lib/ai-tracer.ts
import Langfuse from 'langfuse';

const langfuse = new Langfuse({
  publicKey: process.env.LANGFUSE_PUBLIC_KEY!,
  secretKey: process.env.LANGFUSE_SECRET_KEY!,
  baseUrl: process.env.LANGFUSE_HOST ?? 'https://cloud.langfuse.com',
});

export async function tracedCompletion(
  traceName: string,
  prompt: string,
  fn: () => Promise<string>,
): Promise<string> {
  const trace = langfuse.trace({ name: traceName });
  const generation = trace.generation({
    name: traceName,
    input: prompt,
    model: 'gpt-4o',
  });

  try {
    const output = await fn();
    generation.end({ output });
    return output;
  } catch (err) {
    generation.end({ output: `ERROR: ${(err as Error).message}`, level: 'ERROR' });
    throw err;
  }
}
```

## Fine-tuning vs Prompt Engineering

Decida com base nestas regras — nunca faça fine-tuning só porque parece mais sofisticado:

| Cenário | Decisão | Justificativa |
|---------|---------|---------------|
| Tarefa nova, sem exemplos | Prompt engineering | Fine-tuning requer dados rotulados |
| Output altamente padronizado (< 10 variações) | Prompt com few-shot | Mais barato, mais rápido de iterar |
| Comportamento específico do domínio com 1k+ exemplos | Fine-tuning | Vale o custo quando dados existem |
| Redução de latência/custo em tarefa de alto volume | Fine-tuning com modelo pequeno | GPT-4o-mini fine-tuned > GPT-4o prompt |
| Dados sensíveis que não podem ir ao provider | Modelo local (Ollama) | Privacidade > performance |

**Regra de ouro:** se o problema é resolvível com um bom system prompt + few-shot, não faça fine-tuning.

### Batching de Embeddings

```typescript
// Nunca embedar um item de cada vez em loop — usar batching
async function embedBatch(texts: string[]): Promise<number[][]> {
  // OpenAI suporta até 2048 inputs por chamada
  const batchSize = 100;
  const results: number[][] = [];
  for (let i = 0; i < texts.length; i += batchSize) {
    const batch = texts.slice(i, i + batchSize);
    const response = await openai.embeddings.create({
      model: 'text-embedding-3-small',
      input: batch,
    });
    results.push(...response.data.map(d => d.embedding));
  }
  return results;
}
```

```typescript
async function analyzePdfDocument(pdfBuffer: Buffer): Promise<DocumentAnalysis> {
  // 1. Extração de texto (usar pdf-parse ou similar)
```
